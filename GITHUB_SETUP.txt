Before Pushing to GitHub

1. Verify .env is NOT in repo:
   git status | grep .env
   (Should show nothing)

2. Add .env to gitignore if not already:
   echo ".env" >> .gitignore

3. If .env was accidentally committed, remove it:
   git rm --cached .env
   git commit -m "Remove .env with secrets"

4. Verify no secrets in commit history:
   git log --all --source --remotes -p | grep -i "api_key\|password\|DATABASE_URL"
   (Should return nothing)

5. Tag the release:
   git tag v1.0.0
   git push origin v1.0.0

Initial Clone & Setup for Users

1. Clone repo:
   git clone https://github.com/yourusername/talk-to-your-data.git
   cd talk-to-your-data

2. Create virtual environment:
   python -m venv .venv
   source .venv/bin/activate  # Linux/Mac
   .\.venv\Scripts\Activate   # Windows

3. Install dependencies:
   pip install -r requirements.txt

4. Configure environment:
   cp .env.example .env
   # Edit .env with your real values:
   # - DATABASE_URL: Your PostgreSQL connection
   # - GEMINI_API_KEY: Get from https://makersuite.google.com/app/apikey (optional)
   # - DEV_FALLBACK_MODE: true if no API key

5. Initialize database:
   python populate_db.py

6. Run application:
   python app.py

7. Open browser:
   http://127.0.0.1:5000

GitHub Settings

Recommended
1. Set default branch to "main"
2. Require branch protection:
   - Require pull request reviews (at least 1)
   - Require status checks to pass (if CI/CD set up)
   - Dismiss stale reviews
3. Enable vulnerability scanning (GitHub Advanced Security)
4. Add CI/CD: GitHub Actions workflow for testing
5. Add SECURITY.md with vulnerability reporting guidelines

CI/CD Example (GitHub Actions)

Create .github/workflows/test.yml:

name: Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-python@v2
        with:
          python-version: '3.10'
      - run: pip install -r requirements.txt
      - run: python -m pytest quick_test.py test_validator.py
      - run: python test_edge_cases.py

Troubleshooting for New Users

Database connection fails:
- Check DATABASE_URL in .env
- Ensure PostgreSQL is running: psql --version
- Test connection: psql <your_database_url>

API key errors:
- Set DEV_FALLBACK_MODE=true to run without LLM
- Or get key from https://makersuite.google.com/app/apikey

Port 5000 already in use:
- Change in app.py: app.run(..., port=5001)
- Or kill process: lsof -ti:5000 | xargs kill -9
